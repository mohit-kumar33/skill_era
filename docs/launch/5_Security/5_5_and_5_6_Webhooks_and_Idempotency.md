# 5.5 Webhook Security & 5.6 Idempotency Strategy

**Project Name:** Apex Arena (MVP)  
**Document Owner:** DevSecOps Team  
**Version:** 1.0.0

## 1. Webhook Security Design (Cashfree)

Financial webhooks dictating when to credit a user's wallet with physical funds are the most lethal attack vector. If a malicious actor discovers the `/api/webhooks/cashfree` endpoint and POSTs a fake payload `{"order_id": "123", "status": "SUCCESS", "amount": 90000}`, they could drain the company instantly.

### 1.1 Mitigation: HMAC Verification (Mandatory)
Cashfree signs every webhook payload using the platform's `CASHFREE_SECRET_KEY` and passes the signature in the `x-webhook-signature` header.

**Fastify Webhook Handler Logic:**
1. Check if `x-webhook-signature` exists. If missing, `401 Unauthorized` instantly.
2. Calculate HMAC: `crypto.createHmac('sha256', SECRET).update(parsedTimetamp + rawBody).digest('base64')`. 
3. Compare local HMAC string with provided string using `crypto.timingSafeEqual()` to prevent timing attacks.
4. If identical, process the ledger update. If mismatched, drop the connection immediately.

## 2. Idempotency Strategy

The internet is unreliable. Mobile networks in India routinely drop packets. A user might click "Pay Entry Fee" while navigating a dead zone, receive a network timeout, and frantically click the button three more times.

### 2.1 The Idempotency Key
Every REST API mutation generated by the Next.js client MUST explicitly generate and attach a `X-Idempotency-Key` (a fresh UUIDv4) in the header.

### 2.2 Execution Guarantee
1. Fastify receives the request + `UUID`.
2. Fastify queries Redis (or Postgres MVP Idempotency Table): `SELECT * FROM idempotency WHERE key = 'UUID'`.
3. If Key Exists: Fastify completely halts execution and simply returns the `ResponsePayload` identically cached from the original successful attempt. The database is never mutated again.
4. If Key Missing: Fastify places a lock -> Mutates DB -> Saves the `ResponsePayload` matching the UUID in Redis -> Returns 200 OK.

This guarantees that a user submitting the same ₹50 Entry Fee POST request 15 times simultaneously via Burp Suite only results in a single ₹50 deduction from their wallet.
